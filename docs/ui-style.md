# C2S — UI 风格与设计系统

> macOS 版 "Circle to Search"(净室自研)。本文定义**视觉语言、颜色/字体/间距 token、核心组件规格、动效与无障碍**。实现细节的"怎么做"见 [circle2search-core-mechanisms.md](circle2search-core-mechanisms.md);"做什么"见 [features.md](features.md)。
>
> 平台:macOS 14+,SwiftUI + AppKit 混合,非沙盒。优先用系统材质(NSVisualEffectView / SwiftUI Material)与系统强调色,让它"像系统的一部分",而不是一个突兀的第三方皮肤。

---

## 0. 设计原则

1. **隐形优先(Invisible)**:平时是菜单栏里一个安静的图标;唤起时覆盖层轻、克制,不抢戏。截图冻结要"稳、准、快",不闪。
2. **所见即所得(WYSIWYG)**:选择永远等于用户视觉上圈/划/点到的东西(见核心机制 §6 选词正解)。任何"聪明的自动扩展"都不能违反这一条。
3. **谷歌四色只作点睛(Spectrum as accent)**:蓝/红/黄/绿只出现在"品牌时刻"——全屏微光、加载、四点入场。**正文/UI 一律用系统中性色 + 系统强调色**,避免花哨。
4. **动效有目的(Motion with purpose)**:每个动画都在解释一次状态转移(唤起→冻结、选中→出结果、面板升起)。默认克制;尊重"减弱动态效果"。
5. **原生手感(Native feel)**:材质、圆角、强调色、字体、震动/声音全部走系统约定;能用 SF Symbols 就不自绘图标。

## 1. 颜色 Token

优先**语义 token**,不要在组件里写死 hex。深浅色都必须成立;覆盖层与结果面板以深色为主(叠在截图/网页上),但要跟随系统外观。

### 1.1 中性 / 语义(跟随系统)
| Token | 用途 | 实现 |
|---|---|---|
| `surface.sheet` | 结果面板底、设置面板底 | `NSVisualEffectView(.hudWindow)` 或 SwiftUI `.ultraThinMaterial`;不透明降级 = 深 `#16181C` / 浅 `#F6F6F7` |
| `surface.card` | 结果卡、错误卡 | Material `.regular`;降级深 `#1E2026` / 浅 `#FFFFFF` |
| `surface.control` | 搜索条药丸、按钮底 | 深 `#2A2D35` / 浅 `#ECECEE` |
| `text.primary` | 主文本 | `NSColor.labelColor` |
| `text.secondary` | 次要/元信息 | `NSColor.secondaryLabelColor` |
| `text.tertiary` | 占位/提示 | `NSColor.tertiaryLabelColor` |
| `border.hairline` | 分隔/描边 | `NSColor.separatorColor`(0.5pt) |
| `accent` | 选择高亮、手柄、焦点环、主按钮 | **`NSColor.controlAccentColor`(跟随用户系统强调色)** |
| `scrim` | 冻结覆盖层压暗 | 黑 8%(边缘 ~12% 的极淡径向 vignette,呼应 Android) |

### 1.2 语义状态
| Token | 值(深/浅) | 用途 |
|---|---|---|
| `success` | `#34A853` | 复制成功、连 Wi-Fi 成功 |
| `warning` | `#FBBC05` → 文本用 `#8A5A00` | 权限缺失提示 |
| `danger` | `#EA4335` | 错误卡、失败态 |

### 1.3 品牌光谱(Spectrum — 仅限微光/加载/品牌)
| 名 | Hex | RGB(0–1) |
|---|---|---|
| Blue | `#4285F4` | 0.259, 0.522, 0.957 |
| Red | `#EA4335` | 0.918, 0.263, 0.208 |
| Yellow | `#FBBC05` | 0.984, 0.737, 0.020 |
| Green | `#34A853` | 0.204, 0.659, 0.325 |

> 规则:光谱色**不进入**任何文本/图标/边框/按钮。只用于:全屏四点渐变微光、加载骨架的流光、唤起入场、结果面板首屏 loading 指示。

## 2. 字体

| 角色 | 字体 | 规格 |
|---|---|---|
| 大标题(设置页) | SF Pro Display | 22 / semibold |
| 结果面板搜索条 query | SF Pro Text | 15 / regular,单行截断 |
| 正文 / 卡片 | SF Pro Text | 13–15 / regular,行高 1.4 |
| 元信息 / 来源域名 / 时间 | SF Pro Text | 11–12 / regular / secondary |
| 代码 / 终端类结果 / 条码原文 | **SF Mono** | 12–13,tabular |
| 数字(尺寸/坐标 HUD) | SF Pro Text | tabular-nums |

字重只用 regular / medium / semibold 三档。一律**句子式大小写**,不用全大写。中日韩跟随系统字体回退。

## 3. 间距 / 圆角 / 层级

- **间距刻度**:4 / 8 / 12 / 16 / 24 / 32。组件内用 px 刻度,竖向节奏用 8 的倍数。
- **圆角**:结果/设置面板顶部圆角 `20`;卡片 `12`;控件/按钮 `8`;药丸/胶囊 `height/2`;选择高亮 `3`。单边描边(如左侧强调条)不加圆角。
- **描边**:一律 `0.5pt` hairline;焦点/hover 用强调色 1px 环。
- **窗口层级(NSWindow.level)**:覆盖层 = `.screenSaver`(盖住普通全屏 App;实测若与系统 UI 冲突降到 `.floating`);结果面板锚在覆盖层之上;菜单栏弹层系统管。
- **阴影**:覆盖层内元素**不用投影**(靠材质与描边);唯一例外是结果面板整体升起时的一层柔和 ambient shadow。

## 4. 核心组件规格

### 4.1 冻结覆盖层(Capture Overlay)
- 全屏透明 borderless 窗口,背景 = 截图静帧(**显式 `.frame(点尺寸)`,禁止 `.aspectRatio(.fill)`**,见核心机制 §6 坐标 P0)。
- 叠一层 `scrim`(黑 8% + 边缘极淡 vignette),让"冻结"有轻微沉浸感但不压暗内容。
- 光标切十字 `NSCursor.crosshair`。
- **入场**(120–160ms):截图交叉淡入(实时→冻结,不硬切)+ 一次轻涟漪(见 §6 动效)+ 四点微光 bloom。
- **退场**(120ms):alpha 淡出,恢复上一个 App 焦点。
- Esc / 点空白 = 取消。

### 4.2 四点渐变微光(Spectrum Shimmer)—— 品牌招牌
- 取代原项目"8 光斑公转"。**4 个光谱色点**在画面里做慢速 Lissajous 游走,逐像素按到各点距离做**高斯加权混合**(AE "4-Color Gradient" 数学)。原型见对话里的 `google_4point_gradient_prototype`,落地要点见核心机制 §8。
- 参数默认(2026-07-03 四轮实机调校):speed ≈ 1.7 + 高频微抖;spread idle 0.15 / tracking 0.06,四色点绕笔尖小半径公转(收敛到一点会混成浑浊单色);**idle 用边缘遮罩聚拢到屏幕四周**(smoothstep 0.04→0.26×minDim,中部内容零遮罩 —— 灰字可读性的结构性解法,Android"光晕自边缘扩散"同构),tracking 不受限;低底亮(0.5)+ 陡增辉(k2.2)+ 过饱和 ×1.35 = 色彩分明不泛白纱。
- **两种形态由一个 `trackingAmount` uniform 平滑切换**:idle=全屏四点渐变;涂抹时→收敛成**跟随笔尖的单点辉光**(弹簧跟随)。
- 出现 200ms 渐显 + bloom;涂抹结束转 ambient(降饱和、降亮);退场淡出。
- 遮挡时暂停 Metal 渲染省电;"减弱动态效果"下→静态渐变 + 纯淡入。

### 4.3 选择(Selection)
- **笔刷描边(2026-07-03 参照原版轨迹质感修订)**:白芯 6pt round cap/join + 双层柔光(白 22% 10pt 内晕 + **系统强调色** 16% 16pt 外晕——原版"中心偏白、边缘泛色",mac 端用用户自选 accent 而非写死蓝);笔尖 8pt 白点 + 18pt accent 光斑。
- **词高亮**:被选中的词底色 = `accent` 35% 透明,圆角 3,轻微外扩 2pt(像 Safari 选词)。
- **选择手柄(泪滴)**:`accent` 实心,圆头半径 5,杆宽 2.5,总高 20;起手柄朝上、末手柄朝下;命中半径 20pt。
- **可调矩形(2026-07-03 v3)**:**Liquid Glass 四角括号手柄**(粗臂 7pt、臂长 ≤32、macOS 26+ glassEffect / 旧系统薄材质,外缘 45% 白描边 + 极轻投影)+ 贴边光谱辉光环(**角向渐变持续旋转 24°/s + 呼吸 0.45±0.10**,品牌时刻,减弱动态→静态)+ 框外 16% 压暗;**生成时从框中心弹入**(spring 0.28/0.78 缩放 1.07→1 + 淡入),拖动调整跟手不加动画;**无整圈边框线**(玻璃只在四角,~~hairline~~ 已移除);拖角/拖边实时更新。
- 选择规则严格遵守核心机制 §6 的 WYSIWYG 正解(触碰集 + block 内有界回填,不跨块)。

### 4.4 轻点吸附候选高亮(Tap-to-snap affordance)
- 抓屏后预跑显著性/实例/矩形检测 + 词框(见 features §选择)。
- **hover** 到某个可点区域时,用 `accent` 20% 描边 + 极轻辉光提示"这里可点";轻点即吸附选中(词→整词,物体→物体外接矩形)。
- 复用微光控制器的辉光资源,别单独造轮子。

### 4.5 条码 / 二维码芯片(Barcode chips)
- 二维码框:按内容类型着色的半透明填充(15%,hover 30%)+ 2.5pt 描边;hover 显示"点击打开"提示。
- 下方动作胶囊(chip):`accent` 底 + 白字 + SF Symbol 图标;**点框=打开并退出**,**点胶囊=复制**(变绿勾 + "已复制" 2s)。
- 颜色按类型:URL/邮件=蓝、电话/短信=绿、Wi-Fi=青绿、联系人=橙、位置=红、日程=紫、商品=灰。

### 4.6 结果面板(Result Panel)—— **手机比例悬浮面板**(v3,2026-07-03)
- 形态:**独立悬浮面板,手机屏幕比例**——宽 390pt(iPhone 逻辑宽,谷歌移动版页面在此宽度渲染最佳),高 = 宽 × 19.5/9,两者夹进屏内;默认停靠**屏幕右侧、垂直居中**。
- 交互:顶部**抓手条**可把面板**拖到任意位置**(位置夹在屏内,会话内记住);抓手条右侧 **×**——与 Esc 同义,**退出整个覆盖层**(不是只收面板,2026-07-03 拍板,对齐轻点的重新搜索对称性);拖拽只挂抓手条(不与搜索药丸的 TextField 抢事件)。
- 结构:抓手条(抓手 + ×)→ **搜索条药丸**(查询上下文 = query 文本 **或图搜裁剪缩略图**(40×30 圆角 6,错误卡/重试期间保留)+ 「添加到搜索」输入)→ 结果区(骨架 → WKWebView 无缝替换 / 原生错误卡)。
- 四角圆角 20,`surface.sheet` 材质,hairline 描边 + 柔和 ambient shadow。
- 出现/隐藏:淡入 + 轻微右滑,spring(response 0.4,damping 0.85);减弱动态 → 纯淡入淡出。隐藏时不参与命中测试。
- **WebView 内容处理**:iPhone Safari UA(移动版布局贴合 390pt);注入脚本隐藏 google.com/search 自带顶部搜索条(logo + 表单/图搜缩略图)——查询上下文统一由面板药丸与覆盖层选区承载,筛选 tabs 保留;登录页//sorry/第三方站不动。
- ~~v2:底部三段 detent(peek/half/full)上滑面板~~ —— 全宽横向利用率太低,被否(2026-07-03)。
- **加载态**:先显骨架 +(可选)用几何裁词串**乐观预填** query,后台结果就绪再无缝替换。
- **错误态**:原生错误卡(图标 + 一句话 + 「重试」按钮),**绝不**把错误串当搜索词丢进去搜。

### 4.7 菜单栏图标 + 设置窗口
- 菜单栏:`MenuBarExtra`,图标随屏幕录制权限状态在 `record.circle` / `record.circle.fill` 间切换;菜单含「立即圈选」「设置…」「权限」「退出」。**Info.plist 设 `LSUIElement=YES`**(免启动 Dock 闪一下)。
- 设置窗口:标准 SwiftUI `Settings` 场景,分组:通用(触发方式/热键录制/双击或蓄力开关)、外观(跟随系统/深/浅、减弱动态)、搜索(默认引擎、语言)、权限(状态 + 一键跳系统设置)、关于。**记得给 Settings 场景注入所需的环境对象**(原项目就是这里漏了导致一开设置就崩)。

### 4.8 轻量 HUD / Toast
- 复制成功、权限提示等用一个小型居中/靠上 HUD:material 卡 + SF Symbol + 一行字,1.5–2s 自动消失,spring 入场。

## 5. 动效与时序

| 场景 | 时长 | 曲线 | 备注 |
|---|---|---|---|
| 唤起→冻结 交叉淡入 | 120–160ms | ease-out | 实时→静帧,不硬切;配合预热让空窗趋近 0(见流畅度 Top1) |
| 出现涟漪(RippleShader) | 1.0–1.2s | 线性衰减 | 极轻,作用在截图上 |
| 四点微光渐显 + bloom | 200ms + 800ms bloom | ease-out | idle 形态 |
| 笔尖辉光跟随 | 弹簧,即时跟随 | spring | trackingAmount 0→1 平滑收敛 |
| 词高亮出现 | 120ms | ease | |
| 结果面板升起 | ~380ms | spring(0.4/0.85) | 拖动跟手,下拉关闭 |
| 芯片"已复制"反馈 | 300ms | spring(0.3/0.6) | 变绿勾 + 缩放 1.05 |
| 退场 | 120ms | ease-in | alpha 淡出 |
| 触觉反馈(Force Touch 触控板) | 瞬时 | — | 只在状态确认瞬间:轻点选词/手柄落点/矩形落点=alignment;选区定格/新框=generic;蓄力到点=levelChange;拖动过程绝不连发 |

**减弱动态效果(Reduce Motion)**:关掉四点游走(改静态渐变)、关涟漪与 bloom、面板改纯淡入淡出、笔尖辉光取消跟随。

## 6. 无障碍与输入

- 尊重 `NSWorkspace.accessibilityDisplayShouldReduceMotion` / `ReduceTransparency`(透明降级为实底 token)/ `IncreaseContrast`(描边加深)。
- 全键盘可达:Esc 取消;Tab 在结果面板控件间;⌘C 复制选中;方向键微调可调矩形。
- VoiceOver:覆盖层元素给 label(选中文本、可点物体"图片,点按搜索"、条码类型);结果面板正常读。
- 强调色跟随系统(色盲用户用自己设的强调色);状态**不只靠颜色**——条码类型带图标、错误带图标 + 文案。
- 对比度:文本/背景至少 4.5:1;光谱色不承载信息,故不受对比度约束。
